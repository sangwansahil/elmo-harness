type a prompt. get a fine-tuned small model.

elmo plans the pipeline, finds or generates training data, trains locally on apple silicon or nvidia, evaluates against a public benchmark, and iterates until the model hits the target or the budget runs out. apache-2.0.

install

curl -fsSL https://raw.githubusercontent.com/sangwansahil/elmo-harness/main/install.sh | sh
pip install "elmo-harness[mlx] @ git+https://github.com/sangwansahil/elmo-harness"

apple silicon picks mlx automatically; on nvidia install with the [cuda] extra.

run

elmo init
elmo run examples/function-calling.yaml
elmo serve # localhost:7777

the three bets

01

verifier-first rewards

where a checker exists (function name, json schema, exec, math) elmo uses it as a non-gameable reward. the llm judge is a fallback, not the default.

02

regression suite as artifact

every failure becomes a permanent capability-tagged test case. the model ships with a proof of what it cannot regress on.

03

open trajectory prior

every run publishes (spec, model, recipe, deltas) to a public hf dataset. the planner retrieves from it. closed companies cannot compound across customers.

what works today

  • spec-to-fine-tuned model loop on apple silicon (mlx) or nvidia (unsloth/trl)
  • planner / generator / judge roles, openrouter / groq / anthropic / openai / local providers
  • verifier-first synthetic data with per-row provenance log
  • per-capability deployment gate; regression suite ships with the model
  • rejection-sampling fine-tune (mlx) and grpo (cuda) on the same reward function
  • sqlite completion cache so dev iterations never re-bill
  • local web ui at localhost:7777 with live run view and regression grid

tldr

type "build me a function-calling expert under 2b params that scores 75+ on bfcl". walk away. come back to a model, an eval report, and a regression suite.
